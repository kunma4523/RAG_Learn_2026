# Prompt工程

Prompt（提示词）是与大语言模型交互的核心，好的提示词能显著提升模型输出质量。

## 1. Prompt基本结构

```
┌─────────────────────────────────────────────────────────────┐
│                    Prompt 结构                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [角色定义]                                                 │
│  你是一个专业的AI助手，擅长回答技术问题。                       │
│                                                             │
│  [任务说明]                                                 │
│  请根据以下文档回答用户问题。                                  │
│                                                             │
│  [上下文/参考资料]                                          │
│  文档内容：...                                              │
│                                                             │
│  [输入问题]                                                 │
│  用户问题：什么是RAG？                                       │
│                                                             │
│  [输出格式]                                                 │
│  请用简洁的语言回答。                                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 2. 常用技巧

### 2.1 零样本提示 (Zero-shot)

```python
prompt = "将以下文本分类为正面或负面：这部电影太棒了！"
```

### 2.2 少样本提示 (Few-shot)

```python
prompt = """
请判断情感：

文本：我爱这个产品    情感：正面
类型文本：这东西太差了    情感：负面
文本：还可以         情感：？
"""
```

### 2.3 思维链提示 (Chain-of-Thought)

```python
prompt = """
问题：小明有5个苹果，小红给了他3个，小明吃了2个，小明还剩多少个苹果？

让我们一步步思考：
1. 小明原来有5个苹果
2. 小红给了他3个，5+3=8个
3. 小明吃了2个，8-2=6个
所以小明还剩6个苹果。
"""
```

## 3. RAG中的Prompt模板

### 3.1 标准RAG模板

```python
RAG_PROMPT = """基于以下参考资料回答问题。

参考资料：
{context}

问题：{question}

请根据参考资料回答，如果参考资料中没有相关信息，请如实说明。
"""
```

### 3.2 带思考的模板

```python
RAG_PROMPT_COT = """你是一个专业的问答助手。请根据参考资料回答问题。

要求：
1. 先分析问题
2. 从参考资料中提取相关信息
3. 基于事实回答

参考资料：
{context}

问题：{question}

让我们分析："""
```

### 3.3 对话模板

```python
CHAT_PROMPT = """<|system|>
你是一个友好的AI助手。<|end|>
<|user|>
{query}<|end|>
<|assistant|>"""
```

## 4. Prompt优化技巧

| 技巧 | 说明 | 示例 |
|------|------|------|
| 明确角色 | 指定AI扮演的角色 | "你是一个专业的律师" |
| 分解任务 | 将复杂任务拆分成步骤 | "首先...然后...最后" |
| 提供示例 | 给出期望的输出格式 | 见Few-shot示例 |
| 约束输出 | 指定输出格式 | "用JSON格式返回" |
| 迭代优化 | 根据结果调整Prompt | 不断改进 |

## 5. 常见问题与解决

### 5.1 模型幻觉

```python
# 问题：模型可能编造答案
# 解决：添加约束
prompt = """
请仅根据以下内容回答，不要添加任何额外信息。
如果无法从内容中找到答案，请回答"根据提供的信息无法确定"。
"""
```

### 5.2 输出太长

```python
# 解决：限制输出长度
prompt = "用不超过50个字回答："
```

### 5.3 格式不一致

```python
# 解决：指定明确格式
prompt = """请按以下JSON格式回答：
{"answer": "答案", "confidence": 0.9}"""
```

## 6. 实践建议

1. **从简单开始**: 先用简单Prompt测试，再逐步添加约束
2. **保持一致性**: 在整个应用中保持Prompt风格一致
3. **版本控制**: 记录不同Prompt版本的效果
4. **A/B测试**: 对比不同Prompt的效果
