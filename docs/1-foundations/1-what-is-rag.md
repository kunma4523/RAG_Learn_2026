# 什么是RAG？

RAG（Retrieval-Augmented Generation，检索增强生成）是一种将信息检索与文本生成相结合的技术。

## 1. RAG定义

RAG = 检索(Retrieval) + 增强(Augmentation) + 生成(Generation)

它通过以下方式提升LLM的能力：
1. 从外部知识库检索相关信息
2. 将检索结果作为上下文提供给LLM
3. LLM基于检索上下文生成答案

## 2. 为什么需要RAG？

### 2.1 LLM的局限性

| 问题 | 说明 | RAG解决方案 |
|------|------|-------------|
| 知识过时 | 训练数据有时间限制 | 接入最新知识库 |
| 幻觉 | 模型可能编造事实 | 基于事实检索生成 |
| 无法引用 | 无法提供信息来源 | 直接返回源文档 |
| 长尾知识 | 冷门知识覆盖不足 | 扩展专业领域知识 |

### 2.2 RAG的优势

|------|--------|

| 2021 | DPR、DPR等密集检索出现 |
| 2022 | LangChain、LlamaIndex发布 |
| 2023 | RAG应用爆发 |
| 2024 | Self-RAG、Agentic RAG |
| 2025-2026 | 多模态RAG、长上下文RAG |
